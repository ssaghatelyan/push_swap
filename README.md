# push_swap

## Description

`push_swap` is a sorting algorithm project from the 42 curriculum. The goal is to sort a stack of integers using a limited set of operations, in as few moves as possible, using two stacks — `a` and `b`.

The program reads a list of integers as arguments, outputs the sequence of operations needed to sort stack `a` in ascending order, then exits. If the input is already sorted, nothing is printed.

### Available operations

| Operation | Effect                                |
|-----------|---------------------------------------|
| `sa`      | Swap the top 2 elements of stack a    |
| `sb`      | Swap the top 2 elements of stack b    |
| `ss`      | `sa` and `sb` simultaneously          |
| `pa`      | Push top of b onto a                  |
| `pb`      | Push top of a onto b                  |
| `ra`      | Rotate a upward (top becomes bottom)  |
| `rb`      | Rotate b upward                       |
| `rr`      | `ra` and `rb` simultaneously          |
| `rra`     | Reverse rotate a (bottom becomes top) |
| `rrb`     | Reverse rotate b                      |
| `rrr`     | `rra` and `rrb` simultaneously        |

## Instructions

### Compilation
```bash
make        # build push_swap
make clean  # remove object files
make fclean # remove object files and binary
make re     # full rebuild

Compiled with cc -Wall -Wextra -Werror.

Usage
./push_swap 3 1 2
./push_swap "5 4 3 2 1"
./push_swap 42 -7 0 100 -3
Optional flags
./push_swap --simple   3 1 2
./push_swap --medium   3 1 2
./push_swap --complex  3 1 2
./push_swap --adaptive 3 1 2
./push_swap --bench    3 1 2
Checking correctness
ARG="5 3 1 4 2"
./push_swap $ARG | ./checker $ARG
Bonus (checker)

checker is an additional program that verifies the correctness of the operations generated by push_swap.

Usage
./push_swap 3 2 1 | ./checker 3 2 1

or

ARG="3 2 1"
./push_swap $ARG | ./checker $ARG

You can also input operations manually:

./checker 3 2 1
sa
pb
ra
<Ctrl+D>
Output
OK → stack a is sorted and b is empty
KO → result is incorrect
Error → invalid input or operations
Supported operations

checker supports all operations from the main project:

sa, sb, ss
pa, pb
ra, rb, rr
rra, rrb, rrr

Algorithms
Simple sort — O(n²)

Used for very small inputs (≤ 5 elements).

Medium sort — O(n √n)

Chunk-based sorting using stack b.

Complex sort — O(n log n)

Radix sort based on binary representation.

Adaptive sort

Selects algorithm depending on input size.
